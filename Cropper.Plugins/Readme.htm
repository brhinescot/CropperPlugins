Cropper Plugins v1.3

This is the Cropper Plugins v1.3 release. These plugins are extensions to the Cropper utility, that allow it to send screen shots to various places or online services. Cropper itself, and all these plugins, are free to use.

The Plugins

The plugins included here:

  • AnimatedGif - last updated c. 2007
  • AviFormat - last update c. 2007
  • CountdownPng - allows you to specify a countdown delay before capturing. This gives you time to set up the shot, as is necessary, for example, when the screen shot is a popup menu.
  • SendToEmail - attaches the screen shot to a new message in your default email client
  • SendToFlickr - posts the screenshot on the Flickr web-based photo sharing service
  • SendToImageShack - posts the screenshot on the free ImageShack web-based photo sharing service
  • SendToImgur - posts the screenshot on the free Imgur web-based photo sharing service
  • SendToOneNote - attaches the screenshot to a note in Microsoft OneNote
  • SendToPaintDotNet - opens the screenshot in Paint.NET for editing. (Requires installation of Paint.NET, a free image editing program.
  • SendToS3 - sends the screen shot to Amazon's hosted storage service, S3.
  • SendToTFSWorkItem - posts the screen shot to a new workitem opened in Microsoft's TFS. This works only with TFS 2008.
  • SendToTwitPic - posts the screenshot on the free TwitPic twitter-oriented photo sharing service

To use the plugins

Copy the appropriate DLLs to the cropper plugins directory, typically something like \Program Files\Fusion8Design\Cropper\Plugins. You can copy only the DLLs that you want to use. You should be able to figure out which DLLs provide which function, pretty easily, by their names.

Changes since v1.2 of the Plugins release

  1. The Send-to-TinyPic plugin has been removed. TinyPic is an online image-sharing service, similar to ImageShack or Imgur, except that the terms and conditions for the TinyPic service prohibit using scripts or tools (or anything other than a web browser, I suppose) for uploading images to the service. To discourage the use of tools, TinyPic has changed their transaction flow repeatedly, and recently introduced a Captcha page, which means the old TinyPic plugin no longer works with the current transaction flow required by the TinyPic website. While these are not outright barriers to updating the TinyPic plugin for Cropper to make it work again, they do represent obstacles, both legal and technical. For that reason, I've dropped the TinyPic plugin.
  2. The Send-to-TwitPic plugin is new. It uploads an image to TwitPic, and then posts the image on your twitter feed. Optionally you can provide a Tweet message.
  3. The ImageShack plugin is new.
  4. The Imgur plugin has been updated to use the Imgur API. The v1.2 version of the plugin used HTML screen-scraping, and it broke when Imgur changed their transaction flow. This version should be more reliable.
  5. The ImageShack, Imgur and TwitPic plugins depend on the Microsoft.Http.dll assembly, which is shipped as part of the WCF REST Starter Kit beta 2. The Microsoft.Http.dll assembly must be placed in the Plugins directory of Cropper in order to be able to use either of these plugins.
  6. The S3 plugin has been changed to store its settings in the cropper.config file, in the standard way, via the IConfigurablePlugin interface.
  7. The TFS plugin has been changed to store its settings in the cropper.config file, in the standard way, via the IConfigurablePlugin interface.
  8. The Email plugin now allows you to choose the image format to use. Choices include BMP, JPG, and PNG. Previously you could only use JPG. Or was it PNG? I can't remember.
  9. The source code for all plugins has been fixed up and tweaked. Some of the plugins depended on old, outdated interfaces. I'm surprised they even worked. Now they should all work correctly.

CropperPlugins and Cropper are Separate

Cropper is a screen shot tool. It is produced in the Cropper project hosted on Codeplex.com. The CropperPlugins project is a separately-managed project, also hosted on Codeplex, and it produces these plugins.

The versions and release schedules between these projects are totally independent. The contributors to these projects are different. They are distinct projects.

If you have a problem with cropper, you need to go to the cropper project page for help or support. If you have a problem with one of these plugins, go to the cropperplugins project for support.
